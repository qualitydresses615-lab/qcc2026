# QCC 2026 — Real Backend (Vercel + Postgres/Neon)

This replaces the old single-file demo (which only wrote to a preview
sandbox or in-memory JS) with a genuine backend:

- **Database:** Vercel Postgres (Neon) — one shared, durable, relational store
- **API:** Vercel Serverless Functions (Node.js) under `/api`
- **Frontend:** `public/index.html` — the same design/JS you already
  approved, with the registration form, badge generator, and admin
  dashboard now calling the real API instead of a fake storage layer
- **Sequential Creator IDs:** generated by a real Postgres `SERIAL`
  sequence, so two people registering at the same instant can never
  collide on the same ID
- **Admin auth:** the password lives only in a server-side environment
  variable (`ADMIN_PASSWORD`) — never in the HTML/JS shipped to browsers

## 1. Push this folder to GitHub

```bash
cd qcc-backend
git init
git add .
git commit -m "QCC 2026 with real Vercel Postgres backend"
git branch -M main
git remote add origin <your-empty-github-repo-url>
git push -u origin main
```

## 2. Create the Vercel project

1. Go to https://vercel.com/new and import the GitHub repo you just pushed.
2. Framework preset: **Other** (no build step needed — it's plain
   static HTML + serverless functions).
3. Deploy. It will fail on first load of the admin/registration
   features until the database is attached (steps below) — that's expected.

## 3. Attach a Vercel Postgres (Neon) database

1. In the Vercel dashboard, open your project → **Storage** tab.
2. Click **Create Database** → choose **Postgres** (this is the Neon-backed option).
3. Once created, click **Connect** and select this project — Vercel
   automatically injects `POSTGRES_URL` and related env vars into your
   project. No manual connection string copying needed.

## 4. Run the schema

Open the new database in the Vercel dashboard → **Query** tab (or use
`psql`), paste the contents of `schema.sql` from this repo, and run it.
This creates the `creators` table (with its Creator-ID sequence
starting at 1000) and the `admin_sessions` table.

If you prefer the command line and have `psql` installed:

```bash
psql "$POSTGRES_URL" -f schema.sql
```

(You can copy the exact `POSTGRES_URL` value from Vercel → Storage →
your database → `.env.local` tab.)

## 5. Set the admin password

In Vercel → your project → **Settings → Environment Variables**, add:

| Name | Value |
|---|---|
| `ADMIN_PASSWORD` | choose a strong password — this replaces `quality2026` |

Apply it to Production (and Preview if you want staging admin access
too), then **redeploy** so the function picks it up.

## 6. Redeploy

Trigger a redeploy (Vercel → Deployments → ⋯ → Redeploy, or just push a
new commit). Your site is now live with:

- `/` → the QCC 2026 site
- `POST /api/register` → real registration, atomic sequential IDs
- `GET /api/lookup?mobile=...` → "My Creator Profile" lookup, works from any device
- `GET /api/verify?id=QCC-1000` → public QR-code verification
- `GET /api/count` → public live-counter number only (no personal data)
- `POST /api/admin/login` → server-checked password → session token
- `GET /api/admin/creators` → protected, full list for the dashboard
- `POST /api/admin/update-status` → protected approve/reject

## What changed vs. the old static file

| | Before | Now |
|---|---|---|
| Storage | `window.storage` (Claude preview only) → falls back to a JS variable that resets on refresh | Real Postgres table, shared by every visitor and admin |
| Creator IDs | Manual counter in the same fake storage — two simultaneous registrations could collide | Postgres sequence — atomic, collision-proof |
| Admin password | Sitting in plain text in the page source | Only in a server-side env var, never sent to the browser |
| Cross-device profile lookup | Impossible (each browser only saw its own data) | Works — the badge/profile lookup hits the real database |
| Production readiness | Not usable outside the Claude preview sandbox | Deployable to a real domain today |

## Local development (optional)

```bash
npm i -g vercel
cd qcc-backend
vercel dev
```

`vercel dev` will prompt you to link the project and will pull down the
same `POSTGRES_URL` / `ADMIN_PASSWORD` env vars for local testing.
